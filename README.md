# 📑 Typhoon Invoice AI: Smart Finance Verifier

An intelligent, batch-processing web application designed to automate invoice verification using **Typhoon OCR** and **Typhoon LLM**. This tool bridges the gap between raw document scanning and structured financial approval, featuring a **Human-in-the-Loop (HITL)** interface to ensure 100% data accuracy before exporting to ERP systems.

---

## 🚀 Key Features

* **Dual-Mode Processing**: Seamlessly toggle between **Local (Ollama)** for privacy/cost-saving and **Cloud (OpenTyphoon API)** for high-performance agentic reasoning.
* **Batch Multi-Upload**: Process multiple images (JPG/PNG) or PDFs simultaneously.
* **Automated Verification**: Automatically cross-references extracted Invoice data against a Purchase Order (PO) database to check for amount mismatches or missing records.
* **Human-in-the-Loop (HITL)**: An editable data table allows finance staff to manually correct OCR errors or override system status before finalization.
* **Rate-Limit Protection**: Built-in "Throttling" logic (3.1s delay) and exponential backoff to comply with Typhoon OCR’s **20 req/min** limit, preventing API bans.
* **Excel-Ready Export**: Generates CSV files with `utf-8-sig` encoding to ensure Thai characters display correctly in Microsoft Excel.

---

## 🛠️ Tech Stack

* **Frontend**: [Streamlit](https://streamlit.io/)
* **OCR Engine**: [Typhoon OCR](https://opentyphoon.ai/) (v1.5-3b local / Cloud API)
* **Reasoning Engine**: [Typhoon 2.5 LLM](https://opentyphoon.ai/) (Ollama or Cloud API)
* **Data Handling**: Pandas & Python-Dotenv

🧠 System Logic & Safety
Rate Limiting (Anti-Ban)
The system is optimized for the following Typhoon Cloud limits:

OCR: 20 requests per minute.

LLM: 200 requests per minute.

Strategy: The code implements a mandatory time.sleep() loop to ensure the OCR endpoint is never hit faster than once every 3.1 seconds. If a 429 Too Many Requests error occurs, a 10-second "Cooldown" is triggered before a single retry.

Structured Extraction
The extraction prompt is tuned to return high-precision JSON for seamless data frame integration:

JSON
{
  "invoice_number": "INV-12345",
  "po_reference": "PO-2023-001",
  "total_amount": 10700.00,
  "vendor_name": "บริษัท ABC จำกัด"
}
