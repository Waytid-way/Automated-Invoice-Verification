# โครงการพัฒนาระบบตรวจสอบใบแจ้งหนี้อัตโนมัติสำหรับ Mango Technology

**จัดทำโดย:** ทีมพัฒนา AI Automation  
**วันที่:** 18 เมษายน 2569  
**ประเภทเอกสาร:** ข้อเสนอโครงการ (1 หน้า A4)

---

## 1. ความเป็นมาและปัญหา (Problem Statement)

Mango Technology ประสบปัญหาด้านประสิทธิภาพในกระบวนการตรวจสอบใบแจ้งหนี้จากผู้จัดจำหน่าย (Vendor Invoice) โดยพนักงานบัญชีต้องดำเนินการตรวจสอบด้วยตนเอง ใช้เวลาเฉลี่ย **20–40 นาทีต่อใบแจ้งหนี้ 1 ฉบับ** ส่งผลให้เกิดภาระงานสะสมจำนวนมาก โอกาสเกิดข้อผิดพลาดจากความละเอียดอ่อนของข้อมูล และความล่าช้าในกระบวนการจ่ายเงิน

โครงการนี้เสนอการพัฒนาระบบอัตโนมัติที่ใช้เทคโนโลยี **Typhoon OCR** สำหรับการแปลงเอกสารกระดาษเป็นข้อมูลดิจิทัล และ **Typhoon LLM** สำหรับการวิเคราะห์และสกัดข้อมูลสำคัญ เพื่อลดระยะเวลาและเพิ่มความแม่นยำในการตรวจสอบใบแจ้งหนี้

---

## 2. แนวทางการพัฒนา (Development Approach)

### 2.1 เทคโนโลยีที่ใช้ (Technology Stack)

| ส่วนประกอบ | เทคโนโลยี | หน้าที่ |
|---|---|---|
| OCR Engine | Typhoon OCR | แปลงภาพใบแจ้งหนี้เป็นข้อความดิจิทัล |
| LLM Engine | Typhoon LLM | สกัดและวิเคราะห์ข้อมูลสำคัญจากข้อความ |
| User Interface | Streamlit | สร้างส่วนติดต่อผู้ใช้แบบโต้ตอบ |
| Data Storage | JSON | จัดเก็บข้อมูลใบแจ้งหนี้และผลลัพธ์การตรวจสอบ |

### 2.2 สถาปัตยกรรม Multi-Agent (Swarm Architecture)

ระบบใช้สถาปัตยกรรม Multi-Agent แบบ Swarm ประกอบด้วยเอเจนต์หลายตัวทำงานประสานกัน:

- **OCR Agent** — จัดการการแปลงเอกสาร
- **LLM Extraction Agent** — สกัดข้อมูลสำคัญ
- **3-Layer Duplicate Detection Agent** — ตรวจจับใบแจ้งหนี้ซ้ำซ้อน 3 ระดับ
- **2-Way PO Matching Agent** — ตรวจสอบความตรงกันระหว่างใบสั่งซื้อและใบแจ้งหนี้
- **Error Handler Agent** — จัดการและรายงานข้อผิดพลาด
- **HITL (Human-in-the-Loop)** — ส่งงานที่ต้องการการอนุมัติจากพนักงาน

## 4. ขอบเขตโครงการ (Scope)

### 4.1 ภายในขอบเขต (In Scope)

| รายการ | รายละเอียด |
|---|---|
| การอัปโหลดใบแจ้งหนี้ | รองรับไฟล์ PDF และรูปภาพ (JPG, PNG) |
| การแปลง OCR | แปลงเอกสารเป็นข้อความด้วย Typhoon OCR |
| การสกัดข้อมูล LLM | ดึงข้อมูลสำคัญ (ชื่อผู้จัดจำหน่าย, วันที่, จำนวนเงิน, รายการสินค้า) ด้วย Typhoon LLM |
| การตรวจจับซ้ำซ้อน | ตรวจสอบ 3 ระดับ: ชื่อผู้จัดจำหน่าย + จำนวนเงิน, วันที่ใกล้เคียง, รายการสินค้าซ้ำ |
| การจับคู่ PO | เปรียบเทียบใบสั่งซื้อกับใบแจ้งหนี้ 2 ทาง (ผู้จัดจำหน่าย + จำนวนเงิน) |
| HITL Workflow | ส่งงานที่ผิดปกติหรือต้องการอนุมัติไปยังพนักงาน |
| Export CSV | ส่งออกผลลัพธ์การตรวจสอบเป็นไฟล์ CSV |
| UI Dashboard | ส่วนติดต่อผู้ใช้ด้วย Streamlit สำหรับอัปโหลดและติดตามสถานะ |

### 4.2 อยู่นอกขอบเขต (Out of Scope)

| รายการ | เหตุผล |
|---|---|
| การเชื่อมต่อระบบ ERP | เป็น Phase 2 ที่ต้องพิจารณาแยก |
| การประมวลผลภาษี (VAT/Tax) | ต้องการ Business Rule เฉพาะองค์กร |
| การส่งอีเมลแจ้งเตือนอัตโนมัติ | อยู่ใน Phase 2 |
| การจ่ายเงินอัตโนมัติ (Auto-Payment) | ต้องมีการอนุมัติจากผู้บริหารก่อน |
| การรองรับใบแจ้งหนี้หลายภาษา | Phase 2 (ภาษาไทยเป็นหลัก) |
